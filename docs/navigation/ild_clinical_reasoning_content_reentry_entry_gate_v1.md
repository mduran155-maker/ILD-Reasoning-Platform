## 3. Why v0.20 Is Needed

The previous scaffold and wiring phases created a safe structure.

That structure is necessary, but not sufficient.

A clinically useful ILD platform must help clinicians reason through questions such as:

- Which HRCT pattern is actually carrying interpretive weight?
- Which distribution feature changes the reasoning?
- Which temporal behavior changes the interpretation?
- Which missing evidence prevents stronger interpretation?
- Which mimic remains unresolved?
- Which overlap pattern should not be collapsed prematurely?
- Which source is strong, weak, absent, conflicting, or unsupported?
- Why should confidence remain limited?
- Which MDD questions should be asked next?
- Which report phrase would overclaim?

This is different from a simple differential diagnosis list.

A simple list names possibilities.

The ILD Reasoning Platform should structure reasoning before naming disease.

v0.20 need rule:

> The project must now strengthen clinical reasoning content, not continue expanding generic safety scaffolding indefinitely.

## 7. Evidence-Weighted Reasoning Boundary

v0.20 may prepare evidence-weighted reasoning language.

Allowed:

- evidence supports,
- evidence weakens,
- evidence is missing,
- evidence is conflicting,
- evidence is nonspecific,
- evidence requires review,
- confidence should remain limited,
- mimic remains unresolved,
- overlap persists,
- MDD discussion should address this point.

Blocked:

- probability is X percent,
- diagnosis is confirmed,
- disease is ruled out,
- this is most likely as a final claim,
- treatment should be started,
- follow-up should be scheduled,
- triage is required,
- MDD consensus is generated.

Evidence-weighting rule:

> Evidence weight may guide clinician-facing reasoning, but it must not become unvalidated diagnostic probability.

## 8. Pattern Reasoning Boundary

v0.20 may define pattern reasoning anchors such as:

- fibrotic pattern,
- inflammatory pattern,
- airway-centered pattern,
- nodular pattern,
- cystic pattern,
- mixed pattern,
- overlap pattern,
- indeterminate pattern.

Pattern reasoning may ask:

- What pattern is visible?
- What pattern is missing?
- What pattern is mixed?
- What mimic pattern overlaps?
- What distribution or temporal data would change interpretation?

Pattern reasoning must not say:

- this is UIP,
- this is NSIP,
- this rules out HP,
- this confirms sarcoidosis,
- this excludes infection,
- this is the final diagnosis.

Pattern reasoning rule:

> Pattern language can structure interpretation, but it cannot become autonomous disease naming.

## 9. Mimic and Overlap Boundary

v0.20 must treat mimic and overlap reasoning as core value.

Allowed mimic / overlap functions:

- keep mimics visible,
- identify unresolved overlap,
- mark missing evidence,
- show source conflict,
- prevent premature closure,
- prepare MDD questions.

Blocked mimic / overlap functions:

- autonomous mimic exclusion,
- definitive rule-out,
- final disease ranking,
- hidden probability score,
- treatment pathway selection.

Mimic/overlap rule:

> A mimic remains visible until governed evidence and clinician review justify closure.

- mark missing evidence,
- suggest uncertainty-preserving language,

- generate treatment recommendation,
