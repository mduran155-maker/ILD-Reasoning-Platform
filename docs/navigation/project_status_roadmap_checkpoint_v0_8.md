## 3. Core Platform Identity

The ILD Reasoning Platform is not a disease atlas.

It is not a diagnostic engine.

It is not a treatment engine.

It is not a patient-facing tool.

It is an evidence-governed clinical reasoning platform.

The central principle remains:

> “It teaches clinicians to read change before naming disease.”

The v0.7 principle remains:

> “Differential reasoning is not differential diagnosis authority.”

The v0.8 principle adds:

> “High-risk mimics remain visible without becoming high-risk mimic diagnoses.”

The platform’s current teaching posture is:

- read the pattern,
- compare over time,
- expose missing evidence,
- show source status,
- keep mimics visible,
- preserve competing families,
- allow simultaneous processes,
- distinguish acute overlay from chronic background,
- preserve uncertainty,
- prompt MDD where relevant,
- prompt urgent review where relevant,
- avoid premature disease naming.

## 7. Current Roadmap Position

The platform should not immediately expand into new disease domains.

The next phase should be:

- v0.9 — Reasoning Output Contract / Structured Response Schema

v0.9 should answer:

> How does the platform display reasoning safely, consistently, and source-transparently to a clinician?

v0.9 should not answer:

- What is the diagnosis?
- What treatment should be started?
- What test should be ordered?
- What procedure should be performed?
- What follow-up interval should be scheduled?

## 8. Recommended v0.9 Focus

v0.9 should define a structured output contract containing:

| pattern_family_prompts | show UIP / non-UIP / mimic pattern prompts |

| high_risk_mimic_layer | show high-risk mimics and acute overlays |
