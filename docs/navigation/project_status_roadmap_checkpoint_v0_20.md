## 7. Deterministic Reasoning Engine Strategy

checkpoint-v0.20 records the preferred core architecture:

> The platform should not be pure form-based and should not be pure AI-based.

The preferred architecture is:

> source-governed deterministic reasoning engine.

Core flow:

    structured clinician/user input
    ↓
    deterministic source-governed reasoning rules
    ↓
    reasoning trace
    ↓
    missing evidence
    ↓
    mimic / overlap persistence
    ↓
    non-calibrated evidence weighting
    ↓
    MDD preparation questions
    ↓
    report overclaim guard
    ↓
    clinician review

This architecture is neither:

- a passive checklist,
- nor an autonomous AI diagnosis system.

It is a third path:

> deterministic, auditable, source-governed clinical reasoning support.

Deterministic engine rule:

> The core platform must generate reasoning structure, not free-form clinical conclusions.

---

## 8. Why Not Pure Form-Based

A pure form-based system would allow users to select findings and receive fixed text.

That is safe but too shallow.

Blocked pure-form failure modes:

| Failure Mode | Risk |
|---|---|
| checkbox-only behavior | looks like static checklist |
| no reasoning trace | cannot explain why output appears |
| no mimic persistence | dangerous alternatives disappear |
| no missing evidence logic | unknown fields become invisible |
| no source status | source use cannot be audited |
| no MDD preparation | limited clinical value |
| no report overclaim guard | unsafe wording not detected |

Pure-form boundary:

> Structured input is allowed, but passive checkbox output is not enough.

---

## 9. Why Not Pure AI-Based

A pure AI system would rely on free-text model output and flexible generation.

That may look powerful but creates risks.

Blocked pure-AI failure modes:

| Failure Mode | Risk |
|---|---|
| hallucinated reasoning | unsafe clinical explanation |
| untraceable source logic | poor auditability |
| hidden probability/ranking | authority drift |
| uncontrolled diagnosis language | overclaim |
| treatment/cause leakage | scope violation |
| expensive model dependency | unsustainable core |
| difficult validation | weak academic defensibility |
| hard-to-reproduce outputs | weak deterministic audit |

Pure-AI boundary:

> AI must not be the reasoning authority of the first product layer.
